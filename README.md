# 🍳 Recepto - Збірник рецептів

> Веб-застосунок для зберігання, перегляду та управління кулінарними рецептами. Написаний на чистому HTML, CSS та JavaScript - без жодного фреймворку.

---

## Про проєкт

Я створив цей сайт як повноцінний збірник рецептів з адмін-панеллю. Ідея проста: зручне місце, де можна зберігати улюблені страви з фото, інгредієнтами та покроковим описом приготування. Весь проєкт написаний на ванільному JS без зовнішніх бібліотек - тільки HTML, CSS і JavaScript. Дані зберігаються в `localStorage`, тож сайт повністю працює без сервера.

Я навмисно обрав теплу кольорову палітру (теракот, персик, золото, шавлія) - щоб сайт асоціювався з теплом кухні, а не з холодним корпоративним інтерфейсом.

---

## Як запустити

Жодних залежностей і збірки не потрібно. Треба просто перейти на сайт

При першому запуску автоматично підвантажуються 5 готових рецептів, щоб сайт не був порожнім.

---

## Структура проєкту

```
recipe-book/
├── index.html      # Головна сторінка - сітка рецептів із пошуком і фільтрами
├── recipe.html     # Сторінка окремого рецепту з інгредієнтами та кроками
├── admin.html      # Адмін-панель для CRUD-операцій
├── styles.css      # Всі стилі (CSS-змінні, адаптив, анімації)
└── script.js       # Загальна логіка, дані, робота з localStorage
```

## Функціонал

### Головна сторінка (`index.html`)
- Сітка карток рецептів з фото, назвою, коротким описом, категорією, складністю та часом приготування
- **Миттєвий пошук** по назві рецепту без перезавантаження сторінки
- Фільтрація по категорії (Сніданки, Обіди, Вечері, Десерти, Салати, Супи, Закуски, Напої)
- Фільтрація по складності (1–5 зірок)
- Фільтрація по часу приготування (до 15, 30, 60, 90, 120 хвилин)
- Лічильник знайдених результатів з правильним відмінюванням
- Кнопка скидання всіх фільтрів

### Сторінка рецепту (`recipe.html`)
- Велике фото-заголовок
- Мета-бейджі: категорія, час, складність
- Інгредієнти у прикріпленій лівій колонці (sticky на десктопі)
- Покрокове приготування з інтерактивним відмічанням кроків
- Прогрес-бар виконання
- Святкове повідомлення при завершенні всіх кроків :)
- Кнопка друку рецепту
- Блок "Схожі рецепти"

### **Адмін-панель** `admin.html`
- Авторизація по паролю через `sessionStorage`
- Статистичні картки: кількість рецептів, категорій, середній час і складність
- Таблиця всіх рецептів із пошуком
- Додавання нового рецепту через модальне вікно з валідацією
- Редагування існуючого рецепту
- Видалення з підтвердженням у діалоговому вікні
- Попередній перегляд зображення по URL в реальному часі
- Динамічне додавання/видалення полів інгредієнтів і кроків

## Дизайн і стилі

Я побудував весь дизайн навколо теплої кухонної палітри і двох шрифтів, які добре поєднуються між собою.

### Шрифти
- **Playfair Display** (Google Fonts) - для заголовків. Серифний, елегантний, асоціюється з кулінарними книгами
- **Nunito** (Google Fonts) - для основного тексту. М'який, округлий, добре читається

### **Кольорова палітра**

| Назва | HEX | Де використовується |
|-------|-----|---------------------|
| `--cream` | `#FFF8F2` | Фон сторінок |
| `--terracotta` | `#C8552A` | Акцент, кнопки, посилання |
| `--peach` | `#F2A878` | Hover-ефекти, прикраси |
| `--gold` | `#D4862A` | Зірки рейтингу |
| `--sage` | `#5A7A3C` | Бейджі часу, успіх |
| `--brown-dark` | `#2C1810` | Основний текст |

### CSS-змінні
Я виніс всі константи дизайну в CSS Custom Properties на рівні `:root` щоб легко підтримувати і змінювати тему в одному місці:

```css
:root {
  --terracotta: #C8552A;
  --shadow-md: 0 6px 24px rgba(44, 24, 16, 0.13);
  --r-lg: 24px;
  --trans: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
  /* ...та інші */
}
```

## **JavaScript: архітектура**

Я розділив логіку на три рівні:

**`script.js` - спільне ядро.** Тут: константи (STORAGE_KEY, ADMIN_PASSWORD, CATEGORIES), зразкові рецепти (SAMPLE_RECIPES), всі функції роботи з `localStorage` (`getRecipes`, `saveRecipes`, `addRecipe`, `updateRecipe`, `deleteRecipe`), допоміжні функції (`renderStars`, `formatTime`, `renderRecipeCard`, `escapeHtml`, `getUrlParam`), система Toast-повідомлень і мобільне меню.

**Інлайн-скрипти в кожній сторінці** відповідають лише за логіку конкретної сторінки: фільтрація на `index.html`, інтерактивні кроки на `recipe.html`, CRUD і валідація на `admin.html`.

##  **Зберігання даних**

Всі рецепти зберігаються в `localStorage` за ключем `recipe_book_recipes` у форматі JSON-масиву. Кожен рецепт має таку структуру:

```js
{
  id: "recipe_1748293820000",   // унікальний ID (timestamp)
  title: "Борщ з пампушками",
  description: "Класичний...",
  image: "https://...",
  category: "Обіди",
  difficulty: 4,                // 1–5
  time: 120,                    // хвилини
  ingredients: ["Яловичина — 500 г", ...],
  steps: ["Залийте м'ясо водою...", ...],
  createdAt: 1748293820000,
  updatedAt: 1748293999000      // лише при редагуванні
}
```

## **Адаптивність**

Я дотримувався підходу **mobile-first**. Три основні брейкпоінти:

- **< 480px** - одна колонка, мобільна версія таблиці (картки замість рядків)
- **< 768px** - бургер-меню, дві колонки карток, горизонтальний скрол фільтрів
- **< 1024px** - три колонки карток

## **Доступність**

- Семантична HTML-розмітка (`header`, `nav`, `main`, `footer`, `section`, `article`)
- `aria-label` на всіх інтерактивних елементах
- `aria-live="polite"` на лічильнику результатів
- `role="dialog"` і `aria-modal="true"` на модальних вікнах
- Клавіатурна навігація: `Tab`, `Enter`, `Space`, `Escape`
- `alt`-тексти на всіх зображеннях
- Захист від XSS через функцію `escapeHtml()`

---

## **Безпека**

- Вся вхідна інформація проходить через `escapeHtml()` перед вставкою в DOM
- Пароль адміна перевіряється лише на клієнті (я вважаю, що для демо-проєкту цього достатньо)
- Авторизація зберігається в `sessionStorage` - скидається при закритті вкладки

## **Залежності**

Жодних npm-пакетів. Лише:
- **Google Fonts** (Playfair Display + Nunito) - підвантажуються через `@import`
- **Unsplash** - зображення для зразкових рецептів (через публічне API)
